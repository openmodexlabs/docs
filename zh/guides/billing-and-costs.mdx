## 计费方式

OpenModex 采用按量付费模式。您的账单基于：

- **输入 token** — 您的提示词/消息中的 token
- **输出 token** — 模型生成的 token
- **模型定价** — 每个模型有其自己的每 token 费率

## 定价

OpenModex 以透明的利润率透传供应商定价。查看当前定价：

- [定价页面](https://openmodex.com/pricing)
- [Models API](/zh/api-reference/models-list) — 包含每个模型的定价

## 成本追踪

每个响应都包含成本信息：

```json
{
  "usage": {
    "prompt_tokens": 150,
    "completion_tokens": 50,
    "total_tokens": 200,
    "cost_usd": 0.001
  }
}
```

## 控制面板分析

[控制面板](https://openmodex.com/dashboard)提供：

- **实时消费** — 当前余额和消耗速率
- **按模型明细** — 按模型的成本和用量
- **每日趋势** — 随时间变化的消费情况
- **按密钥分析** — 按 API 密钥追踪成本

## 节省费用

1. **使用成本优化路由** — 让 OpenModex 选择最便宜的模型
2. **启用缓存** — 避免为重复查询付费
3. **选择较小模型** — `gpt-4o-mini` 比 `gpt-4o` 便宜 10 倍
4. **设置 max_tokens** — 限制输出长度以控制成本
5. **使用 BYOK** — 使用您自己的供应商密钥享受批量折扣

## 余额与支付

- 通过**信用卡**（Stripe）或**加密货币**充值
- 在控制面板中查看余额和交易记录
- 设置低余额提醒以避免服务中断
