## 基础 URL

| 环境 | URL |
|------|-----|
| **生产环境** | `https://api.openmodex.com/v1` |
| **沙盒环境** | `https://api-sandbox.openmodex.com/v1` |

使用沙盒环境进行开发和测试——它的行为与生产环境完全一致，但不会产生费用。

所有 API 端点均兼容 OpenAI。如果您已经在使用 OpenAI API，只需更改 base URL 和 API 密钥即可切换。

## 身份验证

在 `Authorization` 头中包含您的 API 密钥：

```
Authorization: Bearer omx_sk_YOUR_API_KEY
```

## 端点

| 方法 | 端点 | 描述 |
|--------|----------|-------------|
| `POST` | `/chat/completions` | 创建聊天补全 |
| `POST` | `/completions` | 创建文本补全（旧版） |
| `POST` | `/embeddings` | 创建嵌入 |
| `GET` | `/models` | 列出可用模型 |
| `GET` | `/models/:id` | 获取模型详情 |
| `GET` | `/models/compare` | 并排对比模型 |

## 请求格式

所有 `POST` 请求接受 JSON 格式的请求体，`Content-Type: application/json`。

## 响应格式

所有响应返回 JSON。成功响应直接包含数据。错误响应遵循以下格式：

```json
{
  "error": {
    "message": "Human-readable error message",
    "type": "invalid_request_error",
    "code": "invalid_api_key"
  }
}
```

## HTTP 状态码

| 状态码 | 描述 |
|------|-------------|
| `200` | 成功 |
| `400` | 请求错误 — 请检查您的参数 |
| `401` | 未授权 — API 密钥无效或缺失 |
| `403` | 禁止 — 权限不足 |
| `404` | 未找到 — 端点或模型无效 |
| `429` | 速率限制 — 请降低请求频率或升级套餐 |
| `500` | 服务器错误 — 请使用退避策略重试 |
| `502` | 网关错误 — 上游供应商错误 |
| `503` | 服务不可用 — 临时中断 |

## OpenModex 响应元数据

每个响应都包含一个可选的 `openmodex` 字段，提供路由和性能元数据：

```json
{
  "id": "chatcmpl-abc123",
  "choices": [...],
  "usage": {...},
  "openmodex": {
    "request_id": "req_abc123",
    "model_used": "gpt-4o-2024-08-06",
    "provider": "openai",
    "cache_hit": false,
    "routing_strategy": "cost_optimized",
    "fallback_used": false,
    "latency_ms": 1250
  }
}
```

## 幂等性

对于 `POST` 请求，您可以传递 `Idempotency-Key` 头以确保相同请求不会被重复处理：

```bash
curl https://api.openmodex.com/v1/chat/completions \
  -H "Authorization: Bearer omx_sk_..." \
  -H "Idempotency-Key: my-unique-key-123" \
  -H "Content-Type: application/json" \
  -d '...'
```

幂等性密钥有效期为 24 小时。
