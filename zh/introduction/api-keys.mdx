## 创建密钥

1. 前往您的 [API 密钥控制面板](https://openmodex.com/dashboard/api-keys)
2. 点击 **Create New Key**
3. 为其设置一个描述性名称（例如 "Production"、"Development"）
4. 立即复制密钥 — 密钥不会再次显示

<Warning>
  您的 API 密钥仅在创建时显示一次。请安全存储。如果丢失，请创建新密钥。
</Warning>

## 管理密钥

在控制面板中，您可以：

- **查看用量** — 查看每个密钥的请求数和费用
- **重命名** — 更新密钥的标签
- **轮换** — 生成新的密钥值，同时保留相同的配置
- **撤销** — 永久禁用密钥

## 密钥轮换

要在不中断服务的情况下轮换密钥：

1. 创建新密钥
2. 更新您的应用程序以使用新密钥
3. 验证新密钥在生产环境中正常工作
4. 撤销旧密钥

## 速率限制

速率限制按 API 密钥应用：

| 套餐 | 每分钟请求数 | 每分钟 Token 数 |
|------|--------------------:|------------------:|
| Free | 30 | 60,000 |
| Pro | 120 | 500,000 |
| Enterprise | 300+ | 自定义 |

当您超出速率限制时，将收到带有 `Retry-After` 头的 `429 Too Many Requests` 响应。

## 使用 BYOK（自带密钥）

OpenModex 还支持使用您自己的供应商 API 密钥进行直接访问：

1. 在控制面板中前往 **Settings → Provider Keys**
2. 添加您的供应商 API 密钥（例如 OpenAI、Anthropic）
3. 密钥使用 AWS KMS 信封加密进行静态加密
4. 当可用时，请求将使用您的供应商密钥进行路由

这使您可以在享受 OpenModex 路由和监控优势的同时使用自己的供应商协议。
