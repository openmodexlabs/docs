## レート制限

レート制限は、サブスクリプションティアに基づいてAPIキーごとに適用されます：

| ティア | リクエスト/分 | トークン/分 |
|------|-------------:|-----------:|
| Free | 30 | 60,000 |
| Pro | 120 | 500,000 |
| Enterprise | 300+ | カスタム |

## レート制限ヘッダー

すべてのレスポンスにレート制限ヘッダーが含まれます：

```
X-RateLimit-Limit: 120
X-RateLimit-Remaining: 115
X-RateLimit-Reset: 1700000060
```

| ヘッダー | 説明 |
|--------|-------------|
| `X-RateLimit-Limit` | ウィンドウあたりの最大リクエスト数 |
| `X-RateLimit-Remaining` | ウィンドウ内の残りリクエスト数 |
| `X-RateLimit-Reset` | ウィンドウがリセットされるUnixタイムスタンプ |

## 429エラーの処理

レート制限を超過すると、`429 Too Many Requests`レスポンスが返されます：

```json
{
  "error": {
    "message": "Rate limit exceeded. Please retry after 2 seconds.",
    "type": "rate_limit_error",
    "code": "rate_limit_exceeded"
  }
}
```

## 自動リトライ

すべての公式SDKは、エクスポネンシャルバックオフで429エラーを自動的にリトライします：

```typescript
const client = new OpenModex({
  apiKey: 'omx_sk_...',
  maxRetries: 3, // default: 2
});
```

リトライの遅延は次の式に従います：`min(500ms × 2^attempt, 5000ms)`

## ベストプラクティス

1. **エクスポネンシャルバックオフを使用** — SDKがこれを自動的に処理します
2. **残りのクォータを監視** — `X-RateLimit-Remaining`ヘッダーを確認してください
3. **ストリーミングを使用** — ストリーミングリクエストはレスポンスの長さに関係なく1リクエストとしてカウントされます
4. **ティアをアップグレード** — 常に制限に達する場合は、アップグレードを検討してください
5. **負荷を分散** — 異なるサービスに複数のAPIキーを使用してください
