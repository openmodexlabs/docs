## キーの作成

1. [APIキーダッシュボード](https://openmodex.com/dashboard/api-keys)にアクセスします
2. **Create New Key**をクリックします
3. わかりやすい名前を付けます（例：「Production」、「Development」）
4. キーをすぐにコピーしてください — 再表示されません

<Warning>
  APIキーは作成時に一度だけ表示されます。安全に保管してください。紛失した場合は、新しいキーを作成してください。
</Warning>

## キーの管理

ダッシュボードから以下の操作が可能です：

- **使用状況の確認** — キーごとのリクエスト数とコストを確認
- **名前の変更** — キーのラベルを更新
- **ローテーション** — 同じ設定を維持しながら新しいキー値を生成
- **無効化** — キーを完全に無効化

## キーのローテーション

ダウンタイムなしでキーをローテーションする方法：

1. 新しいキーを作成します
2. アプリケーションを新しいキーを使用するように更新します
3. 新しいキーが本番環境で動作することを確認します
4. 古いキーを無効化します

## レート制限

レート制限はAPIキーごとに適用されます：

| ティア | 1分あたりのリクエスト数 | 1分あたりのトークン数 |
|------|--------------------:|------------------:|
| Free | 30 | 60,000 |
| Pro | 120 | 500,000 |
| Enterprise | 300+ | カスタム |

レート制限を超過すると、`Retry-After`ヘッダー付きの`429 Too Many Requests`レスポンスが返されます。

## BYOK（自分のキーを持ち込み）の使用

OpenModexは、直接アクセス用に独自のプロバイダーAPIキーの使用もサポートしています：

1. ダッシュボードの**Settings → Provider Keys**にアクセスします
2. プロバイダーAPIキーを追加します（例：OpenAI、Anthropic）
3. キーはAWS KMSエンベロープ暗号化を使用して保存時に暗号化されます
4. 利用可能な場合、リクエストはプロバイダーキーを使用してルーティングされます

これにより、独自のプロバイダー契約を使用しながら、OpenModexのルーティングとモニタリングのメリットを享受できます。
