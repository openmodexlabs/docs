## 課金の仕組み

OpenModexは従量課金モデルを採用しています。課金は以下に基づきます：

- **入力トークン** — プロンプト/メッセージ内のトークン
- **出力トークン** — モデルが生成したトークン
- **モデル料金** — 各モデルには独自のトークン単価があります

## 料金

OpenModexは透明なマージンでプロバイダー料金をパススルーします。現在の料金を確認：

- [料金ページ](https://openmodex.com/pricing)
- [モデルAPI](/ja/api-reference/models-list) — モデルごとの料金を含みます

## コスト追跡

すべてのレスポンスにコスト情報が含まれます：

```json
{
  "usage": {
    "prompt_tokens": 150,
    "completion_tokens": 50,
    "total_tokens": 200,
    "cost_usd": 0.001
  }
}
```

## ダッシュボードアナリティクス

[ダッシュボード](https://openmodex.com/dashboard)では以下が提供されます：

- **リアルタイムの支出** — 現在の残高と消費率
- **モデル別内訳** — モデルごとのコストと使用量
- **日次トレンド** — 時系列での支出推移
- **キー別アナリティクス** — APIキーごとのコスト追跡

## コスト削減

1. **コスト最適化ルーティングを使用** — OpenModexに最も安価なモデルを選択させます
2. **キャッシュを有効化** — 繰り返しクエリの課金を回避します
3. **より小さなモデルを選択** — `gpt-4o-mini`は`gpt-4o`の10分の1の価格です
4. **max_tokensを設定** — 出力長を制限してコストを管理します
5. **BYOKを使用** — ボリュームディスカウントのために独自のプロバイダーキーを持ち込みます

## 残高と支払い

- **クレジットカード**（Stripe）または**暗号通貨**で残高を追加
- ダッシュボードで残高と取引履歴を確認
- サービス中断を防ぐために残高不足アラートを設定
