## 概要

BYOKを使用すると、独自のプロバイダーAPIキー（OpenAI、Anthropicなど）を使用しながら、OpenModexのルーティング、モニタリング、フォールバック、統合課金のメリットを享受できます。

以下の場合に最適です：

- プロバイダーとの**ボリュームディスカウント**やエンタープライズ契約がある場合
- コンプライアンスのために**特定のプロバイダーアカウント**を使用する必要がある場合
- 特定のモデルについてプロバイダーとの**直接課金**を希望する場合

## セットアップ

<Steps>
  <Step title="プロバイダーキーにアクセス">
    [ダッシュボード](https://openmodex.com/dashboard) → **Settings** → **Provider Keys**を開きます。
  </Step>
  <Step title="プロバイダーキーを追加">
    **Add Key**をクリックし、プロバイダーを選択して、APIキーを貼り付けます。
  </Step>
  <Step title="確認">
    OpenModexがテストリクエストでキーを検証します。検証後、そのプロバイダーへのリクエストはあなたのキーを使用します。
  </Step>
</Steps>

## セキュリティ

<Info>
  すべてのプロバイダーキーは**AWS KMSエンベロープ暗号化**（AES-256）を使用して保存時に暗号化されます。OpenModexはプロバイダーキーをログに記録したり公開したりすることはありません。
</Info>

- キーは送信直後に暗号化されます
- 識別用にキーのプレフィックスのみが保存されます（例：`sk-...abc`）
- 復号はリクエスト時にメモリ内でのみ行われます
- キーはログ、アナリティクス、APIレスポンスに含まれることはありません

## BYOKでのルーティングの仕組み

プロバイダーにBYOKキーが設定されている場合：

1. そのプロバイダーへのリクエストは、OpenModexの共有プールではなく**あなたのキー**を使用します
2. プロバイダーから直接課金されます — OpenModexはモデルトークンの料金を請求しません
3. プラットフォーム機能についてはOpenModexのサブスクリプション料金のみお支払いいただきます
4. ルーティング、キャッシュ、フォールバック、モニタリングは通常通り動作します

### 混合構成

一部のプロバイダーにBYOKを設定し、他のプロバイダーにはOpenModexマネージドキーを使用できます：

```
OpenAI     → Your key (BYOK)
Anthropic  → OpenModex managed
Google     → Your key (BYOK)
DeepSeek   → OpenModex managed
```

この場合：
- GPT-4oリクエストはあなたのOpenAIキーを使用（OpenAIから課金）
- Claudeリクエストは OpenModexのキーを使用（OpenModexから課金）
- BYOKとマネージドプロバイダー間のフォールバックはシームレスに動作

## キーの管理

プロバイダーキーダッシュボードから：

| アクション | 説明 |
|--------|-------------|
| **追加** | 新しいプロバイダーキーを追加 |
| **テスト** | テストリクエストでキーの動作を確認 |
| **ローテーション** | キー値を置換（暗号化を更新） |
| **削除** | キーを削除し、リクエストはOpenModexマネージドに戻る |

## 対応プロバイダー

| プロバイダー | キー形式 | BYOKサポート |
|----------|-----------|:---:|

| Anthropic | `sk-ant-...` | 対応 |
| Google | Service Account JSON | 対応 |
| DeepSeek | `sk-...` | 対応 |
| Mistral | `...` | 対応 |
| Qwen | `sk-...` | 対応 |
