title: "チャット補完の作成"

api: "POST /chat/completions"
---

## リクエストボディ

<ParamField body="model" type="string" required>
  使用するモデルID（例：`gpt-4o`、`claude-3.5-sonnet`、`gemini-2.0-flash`）。
</ParamField>

<ParamField body="messages" type="array" required>
  会話を構成するメッセージのリスト。
  <Expandable title="メッセージオブジェクト">
    <ParamField body="role" type="string" required>
      `system`、`user`、`assistant`、または`tool`のいずれか。
    </ParamField>
    <ParamField body="content" type="string" required>
      メッセージの内容。
    </ParamField>
    <ParamField body="name" type="string">
      参加者のオプション名。
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="stream" type="boolean" default={false}>
  `true`の場合、Server-Sent Events（SSE）のストリームを返します。
</ParamField>

<ParamField body="temperature" type="number" default={1}>
  0から2の間のサンプリング温度。低い値はより決定論的になります。
</ParamField>

<ParamField body="max_tokens" type="integer">
  生成するトークンの最大数。
</ParamField>

<ParamField body="top_p" type="number" default={1}>
  Nucleusサンプリングパラメータ。
</ParamField>

<ParamField body="stop" type="string | string[]">
  生成を停止するストップシーケンス。
</ParamField>

<ParamField body="frequency_penalty" type="number" default={0}>
  トークン頻度に対するペナルティ（-2.0から2.0）。
</ParamField>

<ParamField body="presence_penalty" type="number" default={0}>
  トークン存在に対するペナルティ（-2.0から2.0）。
</ParamField>

<ParamField body="tools" type="array">
  モデルが呼び出す可能性のあるツールのリスト（Function Calling）。
</ParamField>

<ParamField body="tool_choice" type="string | object">
  どのツールを呼び出すかを制御します。`auto`、`none`、または特定のツール。
</ParamField>

<ParamField body="response_format" type="object">
  特定の出力形式を強制します（例：`{"type": "json_object"}`）。
</ParamField>

<ParamField body="seed" type="integer">
  再現性のための決定論的サンプリングシード。
</ParamField>

### OpenModex拡張

<ParamField body="routing" type="object">
  このリクエストのインテリジェントルーティングを設定します。
  <Expandable title="ルーティングオブジェクト">
    <ParamField body="strategy" type="string">
      `cost_optimized`、`latency_optimized`、または`quality_optimized`。
    </ParamField>
    <ParamField body="fallback" type="string[]">
      サーバーサイドフォールバックモデルチェーン。
    </ParamField>
    <ParamField body="allow_upgrade" type="boolean">
      同じ価格帯でより優れたモデルへのルーティングを許可します。
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="cache" type="object">
  プロンプトキャッシュを設定します。
  <Expandable title="キャッシュオブジェクト">
    <ParamField body="enabled" type="boolean">
      このリクエストのプロンプトキャッシュを有効にします。
    </ParamField>
    <ParamField body="ttl" type="integer">
      キャッシュの有効期間（秒）。
    </ParamField>
  </Expandable>
</ParamField>

## 使用例

<CodeGroup>
```bash cURL
curl https://api.openmodex.com/v1/chat/completions \
  -H "Authorization: Bearer omx_sk_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4o",
    "messages": [
      {"role": "system", "content": "You are a helpful assistant."},
      {"role": "user", "content": "What is the capital of France?"}
    ],
    "temperature": 0.7,
    "max_tokens": 100
  }'
```

```typescript Node.js
import OpenModex from '@openmodex/sdk';

const client = new OpenModex({ apiKey: 'omx_sk_YOUR_KEY' });

const response = await client.chat.completions.create({
  model: 'gpt-4o',
  messages: [
    { role: 'system', content: 'You are a helpful assistant.' },
    { role: 'user', content: 'What is the capital of France?' },
  ],
  temperature: 0.7,
  max_tokens: 100,
});

console.log(response.choices[0].message.content);
```

```python Python
from openmodex import OpenModex

client = OpenModex(api_key="omx_sk_YOUR_KEY")

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "What is the capital of France?"},
    ],
    temperature=0.7,
    max_tokens=100,
)

print(response.choices[0].message.content)
```
