## Tổng Quan

Streaming cho phép bạn nhận các token phản hồi ngay khi chúng được tạo, thay vì chờ đợi phản hồi hoàn chỉnh. Điều này cải thiện đáng kể độ trễ cảm nhận cho người dùng của bạn.

## Khi Nào Sử Dụng Streaming

- **Giao diện chat** — hiển thị chỉ báo đang gõ và văn bản thời gian thực
- **Phản hồi dài** — không bắt người dùng chờ toàn bộ phản hồi
- **Tạo mã nguồn** — hiển thị mã khi nó đang được viết

## Cách Hoạt Động

1. Đặt `stream: true` trong yêu cầu của bạn
2. OpenModex mở kết nối SSE (Server-Sent Events)
3. Các token được gửi dưới dạng sự kiện `data:` khi chúng được tạo
4. Luồng kết thúc với `data: [DONE]`

## Ví Dụ

<CodeGroup>
```typescript Node.js SDK
const stream = await client.chat.completions.create({
  model: 'gpt-4o',
  messages: [{ role: 'user', content: 'Tell me a story.' }],
  stream: true,
});

for await (const chunk of stream) {
  process.stdout.write(chunk.choices[0]?.delta?.content ?? '');
}
```

```python Python SDK
stream = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "Tell me a story."}],
    stream=True,
)

for chunk in stream:
    content = chunk.choices[0].delta.content
    if content:
        print(content, end="", flush=True)
```

```go Go SDK
stream, _ := client.Chat.Completions.CreateStream(ctx, &openmodex.ChatCompletionParams{
    Model:    "gpt-4o",
    Messages: []openmodex.ChatMessage{{Role: "user", Content: "Tell me a story."}},
})

for chunk, err := range stream.Iter() {
    if err != nil { break }
    fmt.Print(chunk.Choices[0].Delta.Content)
}
```

```tsx React
import { useChat } from '@openmodex/react';

function Chat() {
  const { messages, input, setInput, sendMessage, isLoading } = useChat({
    apiKey: 'omx_sk_...',
    model: 'gpt-4o',
  });

  return (
    <div>
      {messages.map((m) => (
        <div key={m.id}>{m.role}: {m.content}</div>
      ))}
      <input value={input} onChange={(e) => setInput(e.target.value)} />
      <button onClick={() => sendMessage()}>Send</button>
    </div>
  );
}
```

## Streaming + Chuyển Đổi Dự Phòng

Streaming hoạt động liền mạch với chuyển đổi dự phòng. Nếu mô hình chính gặp lỗi trước khi streaming bắt đầu, OpenModex tự động thử lại với mô hình dự phòng:

```typescript
const stream = await client.chat.completions.create({
  model: 'gpt-4o',
  messages: [{ role: 'user', content: 'Hello!' }],
  stream: true,
  routing: {
    fallback: ['claude-3.5-sonnet'],
  },
});
```
