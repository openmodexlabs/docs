---
title: "Créer une complétion de chat"
description: "POST /v1/chat/completions — Créer une réponse de modèle pour une conversation."
openapi: "POST /chat/completions"
---

## Corps de la requête

<ParamField body="model" type="string" required>
  Identifiant du modèle à utiliser (par exemple, `gpt-4o`, `claude-3.5-sonnet`, `gemini-2.0-flash`).
</ParamField>

<ParamField body="messages" type="array" required>
  Une liste de messages composant la conversation.
  <Expandable title="Objet message">
    <ParamField body="role" type="string" required>
      L'un de `system`, `user`, `assistant` ou `tool`.
    </ParamField>
    <ParamField body="content" type="string" required>
      Le contenu du message.
    </ParamField>
    <ParamField body="name" type="string">
      Un nom optionnel pour le participant.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="stream" type="boolean" default={false}>
  Si `true`, renvoie un flux de Server-Sent Events (SSE).
</ParamField>

<ParamField body="temperature" type="number" default={1}>
  Température d'échantillonnage entre 0 et 2. Des valeurs plus basses sont plus déterministes.
</ParamField>

<ParamField body="max_tokens" type="integer">
  Nombre maximum de tokens à générer.
</ParamField>

<ParamField body="top_p" type="number" default={1}>
  Paramètre d'échantillonnage par noyau.
</ParamField>

<ParamField body="stop" type="string | string[]">
  Séquence(s) d'arrêt qui interrompront la génération.
</ParamField>

<ParamField body="frequency_penalty" type="number" default={0}>
  Pénalité de fréquence des tokens (-2.0 à 2.0).
</ParamField>

<ParamField body="presence_penalty" type="number" default={0}>
  Pénalité de présence des tokens (-2.0 à 2.0).
</ParamField>

<ParamField body="tools" type="array">
  Une liste d'outils que le modèle peut appeler (appel de fonctions).
</ParamField>

<ParamField body="tool_choice" type="string | object">
  Contrôle quel outil est appelé. `auto`, `none` ou un outil spécifique.
</ParamField>

<ParamField body="response_format" type="object">
  Forcer un format de sortie spécifique (par exemple, `{"type": "json_object"}`).
</ParamField>

<ParamField body="seed" type="integer">
  Graine d'échantillonnage déterministe pour la reproductibilité.
</ParamField>

### Extensions OpenModex

<ParamField body="routing" type="object">
  Configurer le routage intelligent pour cette requête.
  <Expandable title="Objet routing">
    <ParamField body="strategy" type="string">
      `cost_optimized`, `latency_optimized` ou `quality_optimized`.
    </ParamField>
    <ParamField body="fallback" type="string[]">
      Chaîne de basculement côté serveur.
    </ParamField>
    <ParamField body="allow_upgrade" type="boolean">
      Autoriser le routage vers un meilleur modèle au même niveau de prix.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="cache" type="object">
  Configurer la mise en cache des prompts.
  <Expandable title="Objet cache">
    <ParamField body="enabled" type="boolean">
      Activer la mise en cache des prompts pour cette requête.
    </ParamField>
    <ParamField body="ttl" type="integer">
      Durée de vie du cache en secondes.
    </ParamField>
  </Expandable>
</ParamField>

## Exemple

<CodeGroup>
```bash cURL
curl https://api.openmodex.com/v1/chat/completions \
  -H "Authorization: Bearer omx_sk_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4o",
    "messages": [
      {"role": "system", "content": "You are a helpful assistant."},
      {"role": "user", "content": "What is the capital of France?"}
    ],
    "temperature": 0.7,
    "max_tokens": 100
  }'
```

```typescript Node.js
import OpenModex from '@openmodex/sdk';

const client = new OpenModex({ apiKey: 'omx_sk_YOUR_KEY' });

const response = await client.chat.completions.create({
  model: 'gpt-4o',
  messages: [
    { role: 'system', content: 'You are a helpful assistant.' },
    { role: 'user', content: 'What is the capital of France?' },
  ],
  temperature: 0.7,
  max_tokens: 100,
});

console.log(response.choices[0].message.content);
```

```python Python
from openmodex import OpenModex

client = OpenModex(api_key="omx_sk_YOUR_KEY")

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "What is the capital of France?"},
    ],
    temperature=0.7,
    max_tokens=100,
)

print(response.choices[0].message.content)
```
