## Comment fonctionne la tarification

OpenModex utilise une tarification **à l'usage**. Vous n'êtes facturé que pour les tokens consommés, à des tarifs pass-through transparents.

<Info>
  OpenModex facture le même tarif par token que le fournisseur sous-jacent. Il n'y a pas de majoration cachée sur les coûts des modèles.
</Info>

## Tarification par token

Les coûts des modèles sont transmis aux tarifs des fournisseurs. Consultez les tarifs en temps réel via l'[API Models](/fr/api-reference/models-list) ou le [tableau de bord](https://openmodex.com/dashboard).

### Réponses en cache

La mise en cache des prompts réduit considérablement les coûts :

| | Standard | En cache |
|--|--------:|-------:|
| Tokens d'entrée | 100 % | ~50 % |
| Tokens de sortie | 100 % | 0 $ |
| Latence | Normale | &lt;10ms |

## Suivi des coûts

Chaque réponse API inclut les données de coût :

```json
{
  "usage": {
    "prompt_tokens": 150,
    "completion_tokens": 50,
    "total_tokens": 200,
    "cost_usd": 0.001
  }
}
```

## Solde et paiements

- **Ajoutez des fonds** par carte de crédit (Stripe) ou cryptomonnaie
- **Consultez votre solde** et l'historique des transactions dans le [tableau de bord](https://openmodex.com/dashboard)
- **Configurez des alertes** pour les notifications de solde bas
- **Recharge automatique** disponible sur les niveaux Pro et Enterprise
