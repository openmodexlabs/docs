## 결제 방식

OpenModex는 종량제 모델을 사용합니다. 다음을 기준으로 청구됩니다:

- **입력 토큰** — 프롬프트/메시지의 토큰
- **출력 토큰** — 모델이 생성한 토큰
- **모델 가격** — 각 모델에는 자체 토큰당 요금이 있음

## 가격 정책

OpenModex는 투명한 마진으로 제공업체 가격을 패스스루합니다. 현재 가격을 확인하세요:

- [가격 페이지](https://openmodex.com/pricing)
- [모델 API](/ko/api-reference/models-list) — 모델별 가격 포함

## 비용 추적

모든 응답에는 비용 정보가 포함됩니다:

```json
{
  "usage": {
    "prompt_tokens": 150,
    "completion_tokens": 50,
    "total_tokens": 200,
    "cost_usd": 0.001
  }
}
```

## 대시보드 분석

[대시보드](https://openmodex.com/dashboard)에서 다음을 제공합니다:

- **실시간 지출** — 현재 잔액 및 소진율
- **모델별 내역** — 모델별 비용 및 사용량
- **일별 트렌드** — 시간에 따른 지출 추이
- **키별 분석** — API 키별 비용 추적

## 비용 절감

1. **비용 최적화 라우팅을 사용하세요** — OpenModex가 가장 저렴한 모델을 선택하게 하세요
2. **캐싱을 활성화하세요** — 반복 쿼리 비용을 절감하세요
3. **작은 모델을 선택하세요** — `gpt-4o-mini`는 `gpt-4o`보다 10배 저렴합니다
4. **max_tokens를 설정하세요** — 출력 길이를 제한하여 비용을 관리하세요
5. **BYOK를 사용하세요** — 볼륨 할인을 위해 자체 제공업체 키를 사용하세요

## 잔액 및 결제

- **신용카드** (Stripe) 또는 **암호화폐**로 자금을 추가하세요
- 대시보드에서 잔액 및 거래 내역을 확인하세요
- 서비스 중단을 방지하기 위해 잔액 부족 알림을 설정하세요
