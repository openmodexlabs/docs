## 키 생성

1. [API 키 대시보드](https://openmodex.com/dashboard/api-keys)로 이동하세요
2. **Create New Key**를 클릭하세요
3. 설명이 포함된 이름을 입력하세요 (예: "Production", "Development")
4. 키를 즉시 복사하세요 — 다시 표시되지 않습니다

<Warning>
  API 키는 생성 시 한 번만 표시됩니다. 안전하게 보관하세요. 분실한 경우 새로운 키를 생성하세요.
</Warning>

## 키 관리

대시보드에서 다음을 수행할 수 있습니다:

- **사용량 확인** — 키별 요청 수와 비용 확인
- **이름 변경** — 키의 라벨 업데이트
- **키 교체** — 동일한 설정을 유지하면서 새로운 키 값 생성
- **폐기** — 키를 영구적으로 비활성화

## 키 교체

다운타임 없이 키를 교체하는 방법:

1. 새 키를 생성합니다
2. 애플리케이션을 새 키를 사용하도록 업데이트합니다
3. 프로덕션에서 새 키가 작동하는지 확인합니다
4. 이전 키를 폐기합니다

## 요청 제한

요청 제한은 API 키별로 적용됩니다:

| 티어 | 분당 요청 수 | 분당 토큰 수 |
|------|--------------------:|------------------:|
| Free | 30 | 60,000 |
| Pro | 120 | 500,000 |
| Enterprise | 300+ | 커스텀 |

요청 제한을 초과하면 `Retry-After` 헤더가 포함된 `429 Too Many Requests` 응답을 받게 됩니다.

## BYOK (자체 키 사용) 사용

OpenModex는 직접 접근을 위해 자체 제공업체 API 키를 사용하는 것도 지원합니다:

1. 대시보드에서 **Settings → Provider Keys**로 이동하세요
2. 제공업체 API 키를 추가하세요 (예: OpenAI, Anthropic)
3. 키는 AWS KMS 봉투 암호화를 사용하여 저장 시 암호화됩니다
4. 가능한 경우 요청은 제공업체 키를 사용하여 라우팅됩니다

이를 통해 자체 제공업체 계약을 사용하면서 OpenModex의 라우팅 및 모니터링 혜택을 누릴 수 있습니다.
