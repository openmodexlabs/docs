## API 키

모든 API 요청에는 `Authorization` 헤더에 API 키가 필요합니다:

```bash
Authorization: Bearer omx_sk_YOUR_API_KEY
```

API 키는 [대시보드](https://openmodex.com/dashboard/api-keys)에서 생성하고 관리합니다.

## 키 형식

OpenModex API 키는 다음 형식을 따릅니다:

```
omx_sk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

- `omx_` — OpenModex 접두사
- `sk_` — 비밀 키 식별자
- 이후 랜덤 영숫자 문자열

## 사용법

<CodeGroup>
```bash cURL
curl https://api.openmodex.com/v1/chat/completions \
  -H "Authorization: Bearer omx_sk_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model": "gpt-4o", "messages": [{"role": "user", "content": "Hi"}]}'
```

```typescript Node.js
import OpenModex from '@openmodex/sdk';

// Pass directly
const client = new OpenModex({ apiKey: 'omx_sk_YOUR_KEY' });

// Or use environment variable (auto-detected)
// OPENMODEX_API_KEY=omx_sk_YOUR_KEY
const client = new OpenModex();
```

```python Python
from openmodex import OpenModex

# Pass directly
client = OpenModex(api_key="omx_sk_YOUR_KEY")

# Or use environment variable (auto-detected)
# OPENMODEX_API_KEY=omx_sk_YOUR_KEY
client = OpenModex()
```
</CodeGroup>

## 환경 변수

모든 공식 SDK는 `OPENMODEX_API_KEY` 환경 변수를 자동으로 읽습니다:

```bash
export OPENMODEX_API_KEY=omx_sk_YOUR_KEY
```

## 보안 모범 사례

<Warning>
  클라이언트 측 코드, 공개 저장소 또는 브라우저 요청에 API 키를 노출하지 마세요. API 키에는 결제 권한이 포함되어 있으므로 반드시 비밀로 유지해야 합니다.
</Warning>

- **환경 변수를 사용하세요** — 소스 코드에 키를 하드코딩하지 마세요
- **정기적으로 키를 교체하세요** — 대시보드에서 새 키를 생성하고 이전 키를 폐기하세요
- **개발용과 프로덕션용 키를 분리하세요**
- **권한을 설정하세요** — 가능한 경우 키를 특정 모델이나 요청 제한으로 제한하세요

## 오류 응답

| 상태 | 의미 |
|--------|---------|
| `401 Unauthorized` | API 키가 누락되었거나 유효하지 않음 |
| `403 Forbidden` | 키에 해당 작업에 대한 권한이 없음 |
| `429 Too Many Requests` | 요청 제한 초과 |
