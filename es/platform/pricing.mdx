## Cómo Funcionan los Precios

OpenModex utiliza precios de **pago por uso**. Solo se le factura por los tokens que consume, a tarifas pass-through transparentes.

<Info>
  OpenModex cobra la misma tarifa por token que el proveedor subyacente. No hay recargos ocultos en los costos de modelo.
</Info>

## Precios por Token

Los costos de modelo se pasan a tarifas del proveedor. Consulte los precios en tiempo real a través de la [API de Modelos](/es/api-reference/models-list) o el [Panel de Control](https://openmodex.com/dashboard).

### Respuestas en Caché

El almacenamiento en caché de prompts reduce significativamente los costos:

| | Estándar | En Caché |
|--|--------:|-------:|
| Tokens de entrada | 100% | ~50% |
| Tokens de salida | 100% | $0 |
| Latencia | Normal | <10ms |

## Seguimiento de Costos

Cada respuesta de API incluye datos de costos:

```json
{
  "usage": {
    "prompt_tokens": 150,
    "completion_tokens": 50,
    "total_tokens": 200,
    "cost_usd": 0.001
  }
}
```

## Saldo y Pagos

- **Añada fondos** mediante tarjeta de crédito (Stripe) o criptomonedas
- **Consulte el saldo** y el historial de transacciones en el [Panel de Control](https://openmodex.com/dashboard)
- **Configure alertas** para notificaciones de saldo bajo
- **Recarga automática** disponible en los niveles Pro y Enterprise
