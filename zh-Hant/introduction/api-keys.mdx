## 建立金鑰

1. 前往您的 [API 金鑰控制台](https://openmodex.com/dashboard/api-keys)
2. 點擊 **Create New Key**
3. 為其取一個描述性名稱（例如「Production」、「Development」）
4. 立即複製金鑰 — 此後將不會再顯示

<Warning>
  您的 API 金鑰僅在建立時顯示一次。請安全地儲存它。如果遺失，請建立新的金鑰。
</Warning>

## 管理金鑰

從控制台您可以：

- **查看使用量** — 查看每個金鑰的請求數和成本
- **重新命名** — 更新金鑰的標籤
- **輪換** — 產生新的金鑰值，同時保留相同的設定
- **撤銷** — 永久停用金鑰

## 金鑰輪換

如需在不中斷服務的情況下輪換金鑰：

1. 建立新金鑰
2. 更新您的應用程式以使用新金鑰
3. 驗證新金鑰在正式環境中正常運作
4. 撤銷舊金鑰

## 速率限制

速率限制按 API 金鑰套用：

| 方案 | 每分鐘請求數 | 每分鐘 Token 數 |
|------|--------------------:|------------------:|
| Free | 30 | 60,000 |
| Pro | 120 | 500,000 |
| Enterprise | 300+ | 自訂 |

當超過速率限制時，您將收到帶有 `Retry-After` 標頭的 `429 Too Many Requests` 回應。

## 使用 BYOK（自帶金鑰）

OpenModex 也支援使用您自己的供應商 API 金鑰進行直接存取：

1. 前往控制台中的 **Settings → Provider Keys**
2. 新增您的供應商 API 金鑰（例如 OpenAI、Anthropic）
3. 金鑰使用 AWS KMS 信封加密在靜態狀態下加密儲存
4. 當可用時，請求將使用您的供應商金鑰進行路由

這讓您在使用自己的供應商合約的同時，仍能享受 OpenModex 的路由和監控優勢。
